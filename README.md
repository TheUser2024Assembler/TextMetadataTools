**TME/TMD/TMF (TextMetadataEncoder/TextMetadataDecoder/TextMetadataFinder) - утилиты для стеганографии через невидимые символы.**
*Кодируют/декодируют/находят данные в zero-width символы (U+200B, U+2060).*
**Использование.**
Кодирование (TME.sh)
```shell
# биты -> невидимые символы (сохраняет в 01.txt)
./TME.sh -01 "0101"
# hex -> невидимые символы (сохраняет в 0f.txt)
./TME.sh -0f "deadbeef"
# текст -> невидимые символы (сохраняет в text.txt)
./TME.sh -text "text"
```
*Вывод в терминал показывает результат в кавычках. Файлы 01.txt, 0f.txt, text.txt создаются автоматически.*
Декодирование (TMD.sh)
```shell
# читает 01.txt -> выводит биты
./TMD.sh -01
# читает 0f.txt -> выводит hex
./TMD.sh -0f
# читает text.txt -> выводит текст
./TMD.sh -text
```
*Флаги должны соответствовать флагам, использованным при кодировании.*
Поиск (TMF.sh)
```shell
# ищет в tmf.txt все вхождения zero-width символов, декодирует и выводит с позицией
./TMF.sh
```
*Формат вывода: *`[hex-позиция] декодированные_данные`* (каждый маркер с новой строки). Позиция указана в байтах от начала файла в шестнадцатеричном виде. Данные выводятся как текст (UTF-8). Если данные не являются валидным UTF-8, вывод может быть битым.*
**Пример**
```shell
# кодируем текст
./TME.sh -text "secret"
# находим скрытые данные в любом файле (не только в text.txt)
cat somefile.txt > tmf.txt
./TMF.sh
# выведет: [...] secret
```
**Принцип работы**
- `-01`: 0 -> U+200B, 1 -> U+2060.
- `-0f`: hex -> 4 бита -> 4 символа.
- `-text`: каждый байт -> 8 бит -> 8 символов.
- `TMF.sh`: сканирует файл побайтово, ищет последовательности U+200B/U+2060, декодирует их в байты и выводит как текст.
*Файлы TME/TMD хранят только невидимые символы. Для просмотра используйте *`cat -A`* или декодер. TMF работает с любым файлом, который переименован в *`tmf.txt`*.*
**Внимание**
- Терминал вырезает zero-width символы (они в нём пропадают)
- Передавать через переменные или файлы, не через копирование
- TME/TMD не работают с невалидным ASCII, либо работают не полностью правильно.
- TMF выводит данные как есть; если внутри есть управляющие символы, результат может быть нечитаемым.
